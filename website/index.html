A free terminal coding agent. Your shell, your model.

FreeAgent is a Claude Code-inspired CLI that reads and edits your code, runs commands, and ships projects to GitHub and Vercel. Pick the brain: z.ai's free GLM cloud, Qwen3.6-35B-A3B running 100% locally via Ollama, or any OpenAI-compatible API. Same CLI, one env var.

features

Everything a coding agent should have.

◶

Interactive REPL

Natural-language chat with slash commands, persistent history, and streamed output — just like Claude Code.

⚙

Tool-calling loop

OpenAI-style function calling: the model reads, edits, searches, and runs commands until the job is done.

◌

Sandboxed file ops

read / write / edit / list / grep, all refusing paths that escape your workspace root.

⌘

Shell access

The agent runs builds, tests, and installers for you and reads exit codes + output.

◆

One-prompt GitHub

Ask once — FreeAgent creates a public repo with gh and pushes the initial commit.

▲

One-prompt Vercel

Deploy the current directory to preview or production via the vercel CLI.

◉

Local models · Qwen3.6

Switch to Ollama with one env var and drive FreeAgent with qwen3.6:35b-a3b — a 35B MoE with only ~3B active params. Private, offline, zero cost.

⇌

Swap any OpenAI-compatible API

z.ai, Ollama, OpenAI, or any OpenAI-compatible endpoint — pick one with FREEAGENT_PROVIDER. Same CLI, same tools.

backends

Same agent. Your choice of brain.

FreeAgent's agent loop, tool registry, and prompt layer are identical across all backends. Switch between them with a single FREEAGENT_PROVIDER env var — nothing else changes. Three presets ship out of the box, and adding a fourth is one dict entry.

default

z.ai · GLM-4.5-flash

Free cloud tier. Fastest path from install to first prompt.

  • Cost: $0 — z.ai's free tier, no card needed.
  • Privacy: cloud inference — z.ai sees the request payload.
  • Best for: trying FreeAgent, everyday coding, quick scripts.
export ZAI_API_KEY=your_key
freeagent
local · private

Ollama · Qwen3.6-35B-A3B

A 35B-parameter Mixture-of-Experts model running entirely on your machine.

  • What "35B-A3B" means: 35B total parameters on disk, but only ~3B active per token — frontier-class knowledge at small-model speed.
  • Privacy: 100% local. Code, prompts, tool calls — nothing leaves localhost.
  • Cost: $0 inference. Pay once in disk + VRAM.
  • Hardware: ~22 GB VRAM / unified memory at 4-bit (RTX 4090, A6000, or 32 GB+ M-series Mac). Smaller machines? Swap in a 7B model with FREEAGENT_MODEL.
  • Best for: private / proprietary codebases, long agent sessions, offline work, zero-cost heavy use.
ollama pull qwen3.6:35b-a3b
export FREEAGENT_PROVIDER=ollama
freeagent
frontier

OpenAI · or any compatible API

Drop in OpenAI, Groq, Together, Fireworks, vLLM, LiteLLM, or your own proxy.

  • Cost: whatever your vendor charges per token.
  • Privacy: up to the vendor.
  • Best for: frontier reasoning on a gnarly refactor, or reusing an API you already pay for.
export FREEAGENT_PROVIDER=openai
export OPENAI_API_KEY=sk-...
# optional — any OpenAI-compat endpoint
export FREEAGENT_BASE_URL=https://api.groq.com/openai/v1
export FREEAGENT_MODEL=llama-3.1-70b-versatile
freeagent
install

Two minutes from zero to shipping.

1

Install the CLI

pip install git+https://github.com/vasilysahrai/freeagent.git
2

Grab a free z.ai key

Sign up at z.ai, then export it:

export ZAI_API_KEY=your_key_here
3

Start coding

cd ~/your-project
freeagent

Prefer running models on your own hardware? Install Ollama, then: 

ollama pull qwen3.6:35b-a3b
export FREEAGENT_PROVIDER=ollama
freeagent
how it works

A tight loop between you, the model, and your machine.

┌──────────┐    messages     ┌────────────┐
│   you    │ ───────────────▶│ FreeAgent  │
└──────────┘                 │   agent    │
      ▲                      │            │── tool_calls ─▶  files · bash · grep · gh · vercel
      │    rendered output   │            │◀── results ───
      │                      └────┬───────┘
      │                           │ chat (OpenAI-compatible)
      │                           ▼
      │          ┌──────────────────────────────┐
      │          │   LLM backend (your pick)    │
      └──────────│                              │
                 │  • z.ai    → GLM-4.5-flash   │
                 │  • ollama  → qwen3.6 · …     │
                 │  • openai  → gpt-4o-mini     │
                 │  • …any OpenAI-compat API    │
                 └──────────────────────────────┘

Every turn, FreeAgent sends the running conversation plus a JSON schema for each tool to whichever backend you've configured. The model replies with either natural language (and we stop) or a set of tool calls — FreeAgent executes those locally (files, shell, GitHub, Vercel), appends the results to the conversation, and loops until the model is satisfied.

The backend abstraction is dead simple: every provider is an entry in a PROVIDERS preset dict in freeagent/config.py — base URL, default model, which env var holds the key, whether a key is required. The OpenAI Python SDK handles the transport. Adding a new provider is one dict entry; the agent, tools, and prompts don't change.
